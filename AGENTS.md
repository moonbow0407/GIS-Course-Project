# Agent 编码规范

本文件适用于项目根目录下的全部代码。新增或修改功能时，应优先遵循现有实现和 `README.md`、`docs/` 中的架构约定。

## Agent 工作原则

- **编码前思考**：先阅读相关代码、测试和文档，明确任务范围、假设和验收标准。存在歧义或多种合理解释时先说明，不要默默猜测；发现更简单或更安全的方案时应提出。
- **简洁优先**：只实现用户要求的功能，不为一次性代码增加抽象、配置项或未要求的扩展。能用简单逻辑解决时，不引入额外层次。
- **精准修改**：只修改与任务直接相关的文件和代码；不要顺手重构、格式化或删除既有的无关代码。仅清理由本次改动造成的无用导入、变量和函数。
- **目标驱动**：把“实现功能”转化为可验证结果，例如先写失败测试再修复，重构前后都运行测试；每一步都应有对应检查，直到成功标准满足。

## 项目结构与分层

- `app/domain/`：领域模型和 GIS 状态，只依赖领域/数据处理库，不依赖 PySide6。
- `app/application/`：应用服务、端口、结果对象，负责组织用例和转换异常。
- `app/infrastructure/`：GeoPandas、Rasterio、数据库等外部适配器。
- `app/presentation/`：主窗口、面板、画布和 Qt 渲染器，只处理交互、展示和信号。
- `tests/`：按 `app/` 的模块结构组织测试。
- `main.py`：程序启动与依赖组装，不承载业务逻辑。

依赖方向保持为“界面层 → 应用层 → 领域层 ← 基础设施层”。不要让领域模型直接引用 Qt，也不要在界面控件中直接读写文件或实现空间业务规则。

## Python 编码风格

- 使用 Python 3.10 语法，行宽按 `pyproject.toml` 配置执行（100 字符）。导入由 Ruff 排序。
- 模块、类和公开方法使用简洁的中文文档字符串；复杂算法或不直观的 GIS 坐标处理补充中文注释，说明“为什么”，不要重复代码表面含义。
- 函数、方法和属性尽量完整添加类型注解；局部变量在类型不明显、涉及领域状态或能提升可读性时显式标注。
- 优先使用 `dataclass(frozen=True, slots=True)` 表达不可变结果和领域值对象；集合对外暴露为 `tuple` 或只读结构，避免调用方绕过业务规则修改内部状态。
- 方法保持单一职责。将校验、查找、排序、格式化等可复用逻辑拆成小的私有方法；避免在 Qt 槽函数或应用服务中堆叠过多分支。
- 命名使用清晰的英文，领域术语与已有命名保持一致，例如 `layer_id`、`display_crs`、`WorkspaceSnapshot`、`FeatureId`。
- 不为“以后可能用到”提前增加抽象；只有存在真实变化点时才引入端口、适配器或策略类。

## 注释与文档字符串

- 类、公开方法和关键模块应说明职责；有参数、返回值、异常或状态变化时按现有代码使用“参数 / 返回 / 异常 / 状态变化”分段。
- 注释重点解释坐标系一致性、图层顺序、Qt 与地图坐标转换、空值/无数据处理等业务原因。
- 不保留过时注释、演示数据和与实际功能不符的文档。未实现入口应明确提示“接口已预留”，不得伪造测试结果。

## GIS 与分层约束

- 所有进入 `MapDocument` 的图层必须遵守显示坐标系约束；不能静默叠加 CRS 不一致或坐标系未知的数据。
- 文件读取器负责格式识别、读取和转换为统一领域图层；应用层负责调用读取器并更新工作区；渲染器只负责把快照绘制为 Qt 图元。
- 新增空间分析结果应作为新图层返回或加入工作区，不要无提示覆盖原始图层。
- 文件、数据库、网络等外部资源通过基础设施适配，异常转换为项目应用异常，并向界面提供可理解的中文提示；不要使用裸 `except` 吞掉错误。
- 不在启动时加载测试数据或生成演示结果；路径处理使用 `pathlib.Path`。

## 测试与检查

新增行为先补充测试，测试名称和文档字符串说明行为；重点覆盖正常路径、空数据、非法参数、重复/不存在图层、CRS 冲突和外部读取失败等边界情况。

提交前运行：

```powershell
uv run ruff check .
uv run mypy app
uv run pytest
```

修改界面后，除自动化测试外，应手动确认空工作区、真实图层加载、图层显隐/顺序、坐标与缩放状态，以及错误提示不会导致程序崩溃。

## 修改流程与交付

1. 先阅读相关模块、测试和文档，确认修改所在层级。
2. 用一两句话说明理解、假设、计划和成功标准；不确定时先暂停并请求澄清。
3. 以最小改动实现功能，沿现有端口和结果对象传递数据。
4. 同步补充或更新测试、类型注解和中文文档字符串。
5. 运行格式检查、类型检查和测试，并检查 `git diff` 是否只包含本次任务内容；若环境限制导致未运行，应在交付说明中明确指出。
