# ArcGIS Pro 风格 GIS 工作台界面设计

## 目标

将当前 PySide6 主界面从“菜单栏 + 普通工具栏 + 三栏布局”升级为浅色专业 GIS 工作台。参考用户提供的 ArcGIS Pro 截图，保留现有功能入口和信号连接，重点改善信息层级、工具组织、地图工作区占比和停靠面板的专业感。

## 范围

本次只调整 UI 结构、控件组织、图标表现和 QSS 样式，不改变文件读写、图层模型、地图选择、分析参数校验等业务逻辑。现有未实现功能继续显示“功能开发中”。

## 界面结构

### 1. 快速访问栏

窗口顶部保留系统标题区域，并提供新建/打开、保存、打印、撤销、重做等快速访问操作。控件保持紧凑，避免占用地图工作区高度。

### 2. 功能页签

将现有菜单按工作流组织为“文件、编辑、地图、查询、分析、数据库、帮助”。默认打开“地图”页签。页签只负责切换 Ribbon 内容，不删除原有菜单可访问性。

### 3. Ribbon 工具区

每个页签使用分组工具栏，组内提供图标、文字和必要的快捷提示：

- 文件：新建、打开、保存、另存为、导出
- 导航：平移、缩放、全图、上一视图、下一视图
- 查询：点选、框选、按属性选择、按位置选择、清除选择
- 编辑：新增、修改、删除、撤销、重做、保存编辑
- 分析：缓冲区、叠加分析、线简化、线平滑
- 数据库：连接、导入、加载、导出、断开

工具状态继续由现有 QAction 和 QActionGroup 管理。地图交互工具仍调用 MapCanvas 的 set_map_tool、zoom_in、zoom_out 和 zoom_to_full_extent。

### 4. 左侧内容面板

将 LayerPanel 的标题改为“内容”，增加图层搜索框和紧凑的图层操作条。图层树继续支持可见性、重命名、右键菜单、排序和缩放到图层；当前图层使用浅蓝选中背景。图层图例保留现有自绘逻辑，但统一尺寸和颜色表现。

### 5. 地图工作区

中间区域增加“地图”标签页和新建标签按钮的视觉容器，地图画布保留现有 QGraphicsView 行为。画布使用浅灰工作区背景、白色地图边界和细边框，空状态仍保持可用的空白地图，不伪造真实底图数据。

### 6. 右侧属性与分析面板

RightPanel 使用“属性”和“分析”两个顶部页签。属性页展示选中要素字段和值；无选择时显示明确的空状态。分析页保留现有缓冲区表单，将“运行”作为蓝色主按钮，“重置”作为次按钮，并优化表单间距和分组标题。

### 7. 状态栏

状态栏统一展示坐标、比例尺、当前图层、选中要素和数据源连接状态。数据库状态使用绿色/灰色状态点或标签突出连接状态，分隔线保持轻量。

## 视觉规范

- 主色：ArcGIS 风格的中性蓝 `#2f7de1`，用于页签下划线、选中态、主按钮和焦点边框。
- 背景：窗口和工作区使用 `#f5f6f8`，面板使用白色，地图工作区使用浅灰。
- 边框：统一使用 `#d9dde5`，避免厚重阴影和大面积圆角卡片。
- 文字：主文字 `#243447`，辅助文字 `#6b7480`，禁用文字 `#a4acb7`。
- 控件：工具按钮采用图标上文字下的紧凑布局；普通表单控件统一高度、内边距和焦点样式。
- 尺寸：左侧默认约 300 px，右侧默认约 360 px，中间区域使用伸缩因子占据剩余空间。

## 交互与兼容性

保持现有信号连接和功能键名，尽量通过重新组织 QAction、QToolBar、QTabWidget 和布局实现。窗口缩放时中间地图区域优先伸缩；左右面板可通过 QSplitter 调整宽度。所有现有测试应继续通过，并补充对主窗口关键控件和页签存在性的轻量测试。

## 验收标准

1. 启动后默认显示浅色 ArcGIS Pro 风格工作台，并默认选中“地图”页签。
2. 现有地图导航、选择、图层排序/隐藏/删除、属性展示和缓冲区参数面板仍可用。
3. 顶部工具按功能分组，常用操作不再全部挤在一个平铺工具栏中。
4. 左侧内容、中央地图、右侧属性/分析、底部状态栏的层级清晰，窗口缩放后布局不溢出。
5. QSS 不依赖外部网络资源或未提交的图片素材。
